# 典型系统模式库

4 套典型系统的表结构速查。用户需求匹配时，以此为起点快速出方案，再根据实际需求增减。

## 一、项目管理系统

**适用**：项目管理、任务管理、OKR 跟踪

### 表结构

**表1: 项目表**

| 字段名 | type | 说明 |
|-------|------|------|
| 项目名称 | 1 (文本) | 首字段 |
| 状态 | 3 (单选) | 未开始/进行中/已完成/已搁置 |
| 优先级 | 3 (单选) | P0/P1/P2/P3 |
| 负责人 | 11 (人员) | 项目负责人 |
| 开始日期 | 5 (日期) | |
| 截止日期 | 5 (日期) | |
| 进度 | 2 (进度) | 0-100% |
| 项目描述 | 1 (文本) | 详细说明 |
| 关联任务 | 21 (双向关联) | → 任务表 |

**表2: 任务表**

| 字段名 | type | 说明 |
|-------|------|------|
| 任务名称 | 1 (文本) | 首字段 |
| 所属项目 | 21 (双向关联) | → 项目表 |
| 状态 | 3 (单选) | 待办/进行中/已完成/已取消 |
| 优先级 | 3 (单选) | 紧急/高/中/低 |
| 负责人 | 11 (人员) | |
| 截止日期 | 5 (日期) | |
| 标签 | 4 (多选) | Bug/功能/优化/文档 |
| 备注 | 1 (文本) | |

**表3: 里程碑表**（可选）

| 字段名 | type | 说明 |
|-------|------|------|
| 里程碑名称 | 1 (文本) | 首字段 |
| 所属项目 | 21 (双向关联) | → 项目表 |
| 目标日期 | 5 (日期) | |
| 状态 | 3 (单选) | 未达成/已达成 |
| 说明 | 1 (文本) | |

### 表间关系
```
项目表 ←1:N→ 任务表（双向关联）
项目表 ←1:N→ 里程碑表（双向关联）
```

### 建议视图
- 任务表：看板视图（按状态分组）、按负责人分组视图
- 项目表：进度总览视图

### 建议自动化
- 任务状态变为"已完成"时 → 通知负责人
- 截止日期当天 → 提醒通知

## 三、进销存系统

**适用**：库存管理、商品管理、出入库

### 表结构

**表1: 商品表**

| 字段名 | type | 说明 |
|-------|------|------|
| 商品名称 | 1 (文本) | 首字段 |
| 商品编号 | 1005 (自动编号) | |
| 分类 | 3 (单选) | 按业务自定义 |
| 规格 | 1 (文本) | |
| 单位 | 3 (单选) | 个/箱/kg/... |
| 单价 | 2 (货币) | CNY |
| 安全库存 | 2 (数字) | 最低库存预警值 |
| 关联入库 | 21 (双向关联) | → 入库表 |
| 关联出库 | 21 (双向关联) | → 出库表 |

**表2: 入库表**

| 字段名 | type | 说明 |
|-------|------|------|
| 入库单号 | 1005 (自动编号) | 首字段 |
| 商品 | 21 (双向关联) | → 商品表 |
| 入库数量 | 2 (数字) | |
| 入库日期 | 5 (日期) | |
| 供应商 | 1 (文本) | |
| 经办人 | 11 (人员) | |
| 备注 | 1 (文本) | |

**表3: 出库表**

| 字段名 | type | 说明 |
|-------|------|------|
| 出库单号 | 1005 (自动编号) | 首字段 |
| 商品 | 21 (双向关联) | → 商品表 |
| 出库数量 | 2 (数字) | |
| 出库日期 | 5 (日期) | |
| 领用人/客户 | 1 (文本) | |
| 经办人 | 11 (人员) | |
| 备注 | 1 (文本) | |

**表4: 库存表**（可选，也可用公式在商品表中实现）

| 字段名 | type | 说明 |
|-------|------|------|
| 商品 | 21 (双向关联) | → 商品表 |
| 当前库存 | 20 (公式) | `=SUM(入库数量) - SUM(出库数量)` 需手动配置 |
| 库存状态 | 20 (公式) | `=IF(当前库存<安全库存,"预警","正常")` 需手动配置 |

### 表间关系
```
商品表 ←1:N→ 入库表（双向关联）
商品表 ←1:N→ 出库表（双向关联）
```

### 建议公式（需手动配置）
- 当前库存 = 入库总量 - 出库总量（通过查找引用 + SUM 实现）
- 库存预警 = `IF(当前库存 < 安全库存, "预警", "正常")`

### 建议视图
- 商品表：按分类分组视图
- 入库/出库表：按日期排序视图

## 四、工单系统

**适用**：售后工单、IT 运维、报修管理

### 表结构

**表1: 工单表**

| 字段名 | type | 说明 |
|-------|------|------|
| 工单标题 | 1 (文本) | 首字段 |
| 工单编号 | 1005 (自动编号) | 自定义格式如 WO-001 |
| 类型 | 3 (单选) | 故障/咨询/需求/投诉 |
| 优先级 | 3 (单选) | 紧急/高/中/低 |
| 状态 | 3 (单选) | 待处理/处理中/待确认/已关闭 |
| 提交人 | 11 (人员) | |
| 处理人 | 11 (人员) | |
| 提交时间 | 1001 (创建时间) | |
| 问题描述 | 1 (文本) | |
| 截止时间 | 5 (日期) | |
| 关联处理记录 | 21 (双向关联) | → 处理记录表 |

**表2: 处理记录表**

| 字段名 | type | 说明 |
|-------|------|------|
| 处理说明 | 1 (文本) | 首字段 |
| 所属工单 | 21 (双向关联) | → 工单表 |
| 处理人 | 11 (人员) | |
| 处理时间 | 5 (日期) | |
| 处理结果 | 3 (单选) | 已解决/转交/需跟进/无法解决 |
| 附件 | 17 (附件) | 截图/文档 |
| 备注 | 1 (文本) | |

### 表间关系
```
工单表 ←1:N→ 处理记录表（双向关联）
```

### 建议视图
- 工单表：看板视图（按状态分组）、按优先级分组视图
- 处理记录表：按处理时间排序视图

### 建议自动化
- 新工单提交 → 通知处理人
- 状态变为"已关闭" → 通知提交人
- 截止时间到期且状态非"已关闭" → 升级通知
